# Security Policy

OpenRelief handles high-risk disaster paperwork workflows. Do not post vulnerabilities, privacy leaks, unsafe policy content, secrets, or real survivor PII in public issues.

## Reporting

Use GitHub private security advisory for vulnerabilities or privacy concerns. If private advisory is unavailable, contact repository maintainers through repository owner channels.

## Scope

Security reports may cover:

- uploaded document handling,
- browser-local storage,
- export/import behavior,
- policy pack integrity,
- prompt injection,
- unsafe hosted demo behavior,
- dependency vulnerabilities.

## Data Rule

No real survivor PII in reports, screenshots, fixtures, or reproduction steps. Use synthetic data or describe sensitive values by category.

## Response

Maintainers triage reports using [incident response](docs/incident-response.md). Critical privacy or safety issues can trigger hosted demo takedown before public detail is shared.
