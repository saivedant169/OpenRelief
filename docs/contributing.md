# Contributing

OpenRelief welcomes focused contributions that improve survivor safety, policy grounding, accessibility, or eval coverage.

Before contributing, read the [Code of Conduct](../CODE_OF_CONDUCT.md) and [Synthetic Data License](synthetic-data-license.md).

## Rules

- No real survivor PII in fixtures, issues, screenshots, or tests.
- No policy rule without official source.
- No legal advice examples.
- No live agency submission logic in V1.
- Every new domain behavior needs tests.
- Keep changes small and reviewable.

## Good First Issues

- Add synthetic letter fixture.
- Add source metadata validator.
- Improve checklist copy.
- Add accessibility test.
- Add prompt injection eval case.
